*Языки: [English](README.md) · **Русский** — Инструкция для агента:

## Что это

**llm-wiki** ([оригинальный gist Андрея Карпатого](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f))
— идея хранить документацию кодовой базы как *вики, читаемую LLM*: в каждой
значимой директории лежит `index.md`, страницы
маленькие и об одном, ссылки двусторонние (родитель ⇄ потомок), и у каждой
детали есть ровно одна страница-владелец. Агент идёт по дереву сверху вниз через
`index.md`, а не грепает вслепую.

**llm-wiki-tags** — это то же самое, **плюс тэги**: в самом llm-wiki ничего не
меняется, тэги добавляются сверху.

## Что добавляют тэги

**Тэг** — короткий slug в kebab-case, записанный токеном `@tag:<slug>`. Один и
тот же тэг ставится **и в коде, и в документации**, что создаёт явную
**связь код ⇄ документация**, не зависящую от структуры директорий. С тэгами
можно:

- **искать код и документацию по тэгу** — один поиск возвращает все файлы (код
  или документы), несущие концепт, даже если они разбросаны по дереву;
- **понять, какие тэги есть у части кода** — прочитать тэги вверху файла или
  директории и узнать, в каких сквозных концептах он участвует;
- **понять, какие тэги есть у документа** — то же самое для страницы
  документации;
- держать единый **реестр тэгов** (`docs/tags.md`) с описанием каждого тэга.

Тэги дополняют навигацию через `index.md` (которая идёт по дереву директорий)
второй, ортогональной осью: концепт, охватывающий несколько папок, достижим за
один шаг.

### Формат тэга

Slug — нижний регистр, kebab-case (`[a-z0-9-]+`); один и тот же токен
`@tag:<slug>` ставится и в коде, и в документации.

- **Документация** (`.md`): в **YAML front matter** вверху файла, поле `tags` со
  списком токенов через пробел; для директории — в её `index.md`. Тэг, относящийся
  только к одному разделу, можно вместо этого поставить строкой `@tag:<slug>` в
  теле рядом с ним.
  ```
  ---
  tags: "@tag:payments @tag:retry"
  ---
  ```
- **Код**: комментарий в синтаксисе языка с токеном, поставленный над элементом.
  Тэг может маркировать **пакет**, **файл**, **класс** (или аналог) или
  **метод/функцию**. Правила по языкам — в [`languages/`](languages/index.ru.md):
  [Java](languages/java.ru.md), [Python](languages/python.ru.md),
  [Go](languages/golang.ru.md).
- **Несколько тэгов**: повторить токен через пробел: `@tag:ui @tag:mechanism`.

### Поиск

```bash
# все места (код + документация), где стоит тэг
grep -rn "@tag:<slug>" .

# какие тэги есть у конкретного файла
grep -oE "@tag:[a-z0-9-]+" path/to/file

# все тэги, используемые в репозитории
grep -rhoE "@tag:[a-z0-9-]+" . | sort -u
```

Каждый тэг один раз регистрируется в `docs/tags.md` с однострочным описанием.

## Правила index.md

llm-wiki-tags сохраняет (и делает явными) правила документации из llm-wiki:

1. **В каждой значимой директории есть `index.md`** с **однострочным описанием
   каждого файла и каждой поддиректории** этой папки.
2. Индексы описывают стабильные концепты, а не список изменений. Много маленьких
   страниц лучше одной большой; локальные детали — рядом с кодом, который они
   описывают.
3. **Двусторонняя навигация**: родительские индексы ссылаются на дочерние
   страницы; дочерние — обратно на родительский индекс и на связанные страницы.
4. **У детали одна страница-владелец**; остальные ссылаются на неё — не
   дублировать.
5. **Сначала читай, потом делай**: перед задачей пройди по `index.md` от
   ближайшей директории вниз до кода, который будешь менять.
6. **Обновляй по ходу**: во время или после задачи обнови затронутые `index.md`
   (и тэги) в том же изменении.

## Как подключить

Дайте кодовому агенту ссылку на страницу с инструкцией и попросите поставить:

```
поставь https://github.com/mpashka/llm-wiki-tags/blob/main/INSTRUCTIONS.ru.md
# или по-английски:
install https://github.com/mpashka/llm-wiki-tags/blob/main/INSTRUCTIONS.md
```

Агент читает [`INSTRUCTIONS.ru.md`](INSTRUCTIONS.ru.md) (или
[`INSTRUCTIONS.md`](INSTRUCTIONS.md)) и настраивает в текущем репозитории дерево
`index.md`, механизм тэгов и реестр `docs/tags.md`, а также фиксирует соглашение
в гайде для агентов, чтобы будущие агенты его соблюдали.
